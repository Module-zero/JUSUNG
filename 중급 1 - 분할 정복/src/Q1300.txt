크기가 n×n 인 2차원 배열 A 가 있다
배열에 들어있는 수 A[i][j] = i×j 이다
이 수를 1차원 배열 B 에 넣으면 B 의 크기는 n×n 이 된다
B 를 오름차순 정렬했을 때, B[k] 를 구해보자
배열 A 와 B 의 인덱스는 1부터 시작한다
(1 <= n <= 100000, 1 <= k <= 10억)

배열 A, B 의 최대크기 : int 방 10억개
따라서 배열 A, B 를 직접 생성하는 것은 불가능하다
B 를 직접 정렬하고, 인덱스 k 에 직접 접근하는 것 역시 불가능하다

이분탐색에서 left = 1, right = n*n 으로 시작한다
인덱스 k 는 문제에서 주어진다. 이 값을 어떻게 활용해야할까?
k 를 이분탐색의 조건으로 활용하는 것은 맞을 것이다

2차원 배열에 어떤 규칙이 있을까? 단번에 인덱스 k 의 값을 구하는 공식 같은 것은 없을 것이다
